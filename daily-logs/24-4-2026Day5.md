##  Objective
Evaluation of visual editor libraries (**Puck** vs. **Craft.js**) to satisfy the requirement of "splitting config from representation" for our platform. The goal is to replicate a Shopify-like theme architecture where the UI is a visual layer over a portable JSON schema.

---

##  Core Concept: "Config vs. Representation"
To ensure our Rust backend can independently store, validate, and version data, we must strictly separate the data from the editor:

* **Representation (The Data):** A pure JSON tree defining sections, blocks, and props (similar to Shopify’s `sections/*.json`).
* **Config (The Blueprint):** The React components and field schemas that interpret the JSON.
* **Requirement:** The internal state must never be trapped in React; it must remain **JSON-in, JSON-out**.

---

##  Tool Evaluation

### 1. Puck (Recommended)
* **Architecture:** Native "JSON-first" design. The "config vs. representation" pattern is the core engine, not an afterthought.
* **Data Model:** The schema naturally mirrors Shopify’s `sections` and `blocks` structure.
* **Team Velocity:** Ideal for a 2-person Front-End team. It allows us to focus on building high-value UI components rather than engineering a low-level drag-and-drop tree engine.
* **Extensibility:** We can maintain high velocity now and fork the Puck repo later to add custom schema validations or multi-store features.

### 2. Craft.js (Overkill for V1)
* **Architecture:** A powerful state-tree library for nodes and props, but lacks a predefined JSON schema or UI.
* **Drawback:** Requires building the entire editor UI, serialization logic, and Shopify-style wrappers from scratch.
* **Verdict:** **Unnecessary for MVP.** We can revisit this for advanced internal dev-tooling if we outgrow Puck's internals in the future.

---

##  Infrastructure & Storage Impact
* **Footprint:** JSON files are negligible (~1–10 KB per page). Storage is a non-issue.
* **Storage Strategy:** The Rust backend will store one JSON blob per theme/store. This facilitates trivial versioning (e.g., `theme_v1.json`) and caching.
* **Performance:** The bottleneck will be React rendering (images/JS), not the JSON retrieval or parsing.
