# Architecture MVP: Integrating Puck Visual Editor with Shopify Liquid Templates

**Date:** April 30, 2026
**Focus:** Front-End Architecture, Server-Side Rendering (SSR) Constraints, and React-to-HTML Serialization.

## Executive Summary

This repository contains the Minimum Viable Product (MVP) designed to resolve a core architectural mismatch between our visual building tooling and our deployment environment. The objective is to utilize a React-based visual editor (`@measured/puck`) to author content, while adhering to the strict Server-Side Rendering (SSR) and pure HTML requirements of a purchased Shopify Liquid template. 

This MVP proves the technical feasibility of constructing components visually in React, capturing the state as JSON, and serializing that state into a static HTML string that can be seamlessly injected into Shopify's ecosystem.

## Technical Stack

*   **Build Tool:** Vite
*   **Core Library:** React, TypeScript
*   **Visual Editor:** `@measured/puck`
*   **Styling:** Tailwind CSS
*   **Serialization:** `ReactDOMServer`

## MVP Implementation & Division of Labor

The MVP is structured as a split-screen sandbox to independently test the visual building experience and the subsequent HTML compilation. 

### Part 1: The Editor Engine
This module handles the authoring experience and state generation.
*   **Component Configuration (`puckConfig.tsx`):** Defined a highly specific component schema (`ShopifyHeroBanner`) to mirror the structural requirements of the target Shopify template.
*   **Visual Interface:** Implemented the Puck editor interface, restricting the user to the predefined schema.
*   **State Capture:** Configured the editor to listen for state changes and capture the resulting `appState.data` payload.

### Part 2: The HTML Compiler
This module handles the serialization and validation of the output.
*   **React-to-HTML Serialization:** Utilized `ReactDOMServer.renderToString()` to process the React nodes generated by Puck into a raw, static HTML string.
*   **Metadata Extraction:** Built logic to parse the `root` object of the Puck JSON payload to extract standard SEO metadata (Title, Description).
*   **Live Preview & Output:** Engineered a dashboard that renders the pure HTML directly to the DOM for visual validation, alongside a code-export block displaying the raw string required for Shopify.

## Data Flow Architecture: JSON to HTML

The core challenge of this MVP is managing the transformation of interactive React state into static markup. The data pipeline operates as follows:

1.  **State Mutation (Authoring):** The user modifies fields within the Puck UI (e.g., changing text or background colors). Puck updates its internal Zustand state manager.
2.  **Payload Extraction (JSON):** Upon modification or publication, Puck outputs a serialized JSON object representing the document tree. This object contains two primary keys: `content` (an array of component objects with their specific props) and `root` (global document settings).
3.  **State Lifting:** The JSON payload is passed upward to a shared parent container (`App.tsx`), ensuring both the Editor module and the Compiler module share a single source of truth.
4.  **Component Hydration:** The JSON payload is passed into Puck's headless `<Render />` component, which reconstructs the React Virtual DOM based on the provided `puckConfig`.
5.  **Serialization (HTML Generation):** Instead of mounting this Virtual DOM to the browser, the application executes `renderToString(<Render config={config} data={payload} />)`. This forces React to synchronously evaluate the component tree and output standard HTML text.
6.  **Final Output:** The resulting HTML string is completely decoupled from React's client-side JavaScript bundle, making it lightweight, universally compatible, and ready for ingestion by a Shopify `.liquid` template.
